Support for polymorphism - A test drive

One of the major short-comings of Castor JDO always has been missing support for polymorphism. After the 0.9.6 release back in Februray, which mainly focused on bug fixing, we decided to tackle this missing feature and start work for adding support for polymorphism to Castor JDO. For a complete history of the work done so far and the tests conducted, please have a look at bug 1881.

As of last week, most of the work (originally initiated by myself) has been finished, and we are now in a position where we'd like to ask you, our users, for feedback. In other words, we think that the current implementation is stable enough to make it available as described below. But as we want to get this new feature as stable as possible before heading for a new release (most likely 0.9.7, but maybe 1.0), we now need feedback from our users.

To facilitate this, I've made available binaries here. Please use the source distribution if possible, as it contains the tests mentioned above as well.

Mailing list changes

This is to announce that we finally moved our mailing lists to the Codehaus. As part of this move, we moved your subscriptions to spare you the work of having to re-subscribe. If you feel that you are subscribed to one of the mailing lists by fault, please use the below addresses to unsubscribe.

As of now, please use the following email addresses:

* User mailing list

To post a message: user@castor.codehaus.org
To subscribe: user-subscribe@castor.codehaus.org
To unsubscribe: user-unsubscribe@castor.codehaus.org
Archive: http://archive.castor.codehaus.org/user

* Announcement mailing list

To post a message: announce@castor.codehaus.org
To subscribe: announce-subscribe@castor.codehaus.org
To unsubscribe: announce-unsubscribe@castor.codehaus.org
Archive: http://archive.castor.codehaus.org/announce

* Development mailing list

To post a message: dev@castor.codehaus.org
To subscribe: dev-subscribe@castor.codehaus.org
To unsubscribe: dev-unsubscribe@castor.codehaus.org
Archive: http://archive.castor.codehaus.org/dev

For a period of 14 days, we'll be monitoring the old lists at Exolab, mainly referring you to the lists mentioned above. After this period, we will turn off these lists at our discretion. As for mailing lists archives, please note that at the moment, the archives mentioned above don't hold any messages from the old lists, but we are working on a solution. As such, messages from the lists at Exolab will be available at the usual sites.
